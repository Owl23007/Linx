# Linx 信息架构与主流程

最后更新：2026-03-25

## 一级页面

### 1. 登录页

目标：

- 进入系统
- 选择服务节点
- 管理本地保存账号

当前状态：

- 已有较完整实现
- 仍以“账号登录”为核心，不需要大改

### 2. 联机首页

目标：

- 创建房间
- 加入房间
- 展示最近房间
- 展示最近联机伙伴
- 展示当前网络状态和最近一次错误

当前代码入口：

- `front/src/views/pages/lobby/index.vue`

### 3. 房间页

目标：

- 展示成员列表
- 展示房主和当前房间状态
- 展示我的虚拟 IP
- 展示成员连接质量
- 提供复制 IP、重新连接、启动游戏、打开诊断

### 4. 诊断抽屉 / 诊断页

目标：

- 告诉用户哪里失败
- 告诉用户下一步怎么修
- 集中呈现权限、驱动、P2P、中继、RPC、节点可达性等问题

## 核心实体

### user

- 账号身份
- 昵称
- 登录态

### device

- 当前设备名
- 当前系统环境
- EasyTier 运行状态

### room

- 房间 ID
- 房间码
- 房间名称
- 房主
- 服务节点
- 当前网络参数

### roomMember

- 用户 ID
- 设备 ID
- 房间角色
- 在线状态
- 可达状态
- 虚拟 IP

### peerConnection

- 连接模式：P2P / Relay
- 延迟
- 丢包
- 最近错误

### gameProfile

- 游戏名称
- 推荐启动方式
- 默认端口或预设

## 连接状态中心

建议新增统一的连接状态中心，而不是把状态散在页面组件中。

推荐聚合字段：

- EasyTier 是否运行
- 当前房间
- 当前房间成员列表
- 当前设备虚拟 IP
- Peer 列表
- Route 列表
- 连接模式
- 最近诊断结果
- 最近错误

建议状态来源：

- Electron IPC：EasyTier 进程状态、Peer、Route
- 前端页面状态：当前房间、房间码、最近操作
- 服务端：房间元数据、成员关系、邀请与持久化信息

## 关键流程

### 创建房间

```text
联机首页
-> 点击创建房间
-> 选择或生成房间参数
-> 启动 EasyTier
-> 创建房间记录
-> 进入房间页
-> 分享房间码
```

### 加入房间

```text
联机首页
-> 输入房间码
-> 获取房间元数据
-> 启动 EasyTier 并加入网络
-> 展示成员与连接状态
-> 复制 IP / 启动游戏
```

### 失败诊断

```text
连接失败
-> 打开诊断抽屉
-> 展示失败类型
-> 给出修复动作
-> 重试连接
```
