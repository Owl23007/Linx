# Linx 交互文案规范

最后更新：2026-03-25

## 文案目标

- 让普通玩家能读懂
- 保留一定技术准确性
- 尽量避免 IM 产品语言
- 在失败时直接给动作建议

## 语气原则

- 直接，不绕弯
- 优先说结果，再说原因
- 优先说用户下一步能做什么
- 技术词只在必要时出现

## 推荐术语映射

| 旧说法 | 新说法 |
| --- | --- |
| 联系人 | 联机伙伴 |
| 会话 / 聊天 | 房间 |
| 在线 | 可联机 |
| 离线 | 不可达 |
| 发消息 | 发送邀请 |
| 群聊 | 联机房间 |
| 服务器地址 | 服务节点 |
| 节点 | 设备 |
| 隧道已建立 | 联机已就绪 |
| Relay | 中继模式 |
| NAT 失败 | 未能建立直连 |
| TUN/驱动错误 | 需要系统授权以建立虚拟网线 |

## 状态文案规范

### 运行状态

- EasyTier 已启动
- 联机已就绪
- 正在连接房间
- 正在重试
- 当前使用中继模式
- 未检测到虚拟网卡

### 成员状态

- 可联机
- 正在连接
- 不可达
- 延迟较高
- 已离开房间

### 操作文案

- 创建房间
- 加入房间
- 复制虚拟 IP
- 复制房间码
- 重新连接
- 打开诊断
- 启动游戏

## 错误文案模板

推荐格式：

```text
[结果] + [原因] + [下一步动作]
```

示例：

- 未能建立直连，当前已切换到中继模式。你可以继续联机，或检查本机网络权限后重试。
- 未检测到虚拟网卡，可能缺少系统授权。请先完成授权，再重新连接。
- 服务节点不可达，请检查服务节点地址或稍后重试。
- 当前房间连接已中断，请重新加入房间。

## 禁止倾向

避免这些表达：

- 会话
- 消息中心
- 已读
- 未读
- 聊天对象
- 群成员消息
