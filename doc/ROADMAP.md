# Linx 路线图

最后更新：2026-03-25

## M1：联机闭环

目标：让用户完成一次从登录到成功联机的最短路径。

范围：

- 登录
- 联机首页
- 创建房间 / 加入房间
- 房间码分享
- 本地拉起 EasyTier
- 成员列表
- 复制虚拟 IP

验收标准：

- 用户可独立完成创建房间和加入房间
- 房间内可看到成员和自己的虚拟 IP
- 失败时至少有一个明确错误出口

## M2：连接可见性与诊断

目标：让用户知道当前连接是否可靠，以及失败时能怎么修。

范围：

- 连接状态中心
- P2P / 中继模式标识
- 重连
- 基础诊断
- 权限修复入口

验收标准：

- 用户能区分是否直连
- 用户能看到最近错误和推荐动作
- 常见失败可回到主流程

## M3：游戏预设与效率提升

目标：减少每次联机前的重复操作。

范围：

- 游戏预设
- 一键启动游戏
- 最近房间
- 最近联机伙伴

验收标准：

- 高频联机流程可在首页直接复用
- 至少一个游戏预设可以跑通

## M4：长期能力

目标：补齐可持续运营和管理能力，但不破坏主流程简洁性。

范围：

- 真正的好友关系
- 房间持久化
- 多房间
- 更完整的设备管理

验收标准：

- 多设备、多房间、多次联机场景可维护
- 社交能力不压过联机主流程

## 当前建议顺序

1. 先完成 M1 文档与页面基线
2. 再做 M2 的连接状态中心和诊断
3. M3、M4 在主流程稳定后推进
